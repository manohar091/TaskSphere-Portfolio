## üìù **Screenshot Guidelines**

### **Quality Standards**
- **Resolution**: Minimum 1920x1080 for web, native resolution for mobile
- **Format**: PNG for UI elements, JPG for photos
- **Compression**: Optimize for web without quality loss
- **Naming**: Use descriptive, kebab-case file names

### **Content Guidelines**
- Use realistic test data (no Lorem Ipsum)
- Show actual functionality, not mock-ups
- Include diverse user scenarios
- Demonstrate key features and workflows

### **Privacy & Security**
- Remove or anonymize sensitive information
- Use test accounts and sample data
- Blur out personal information
- Ensure no production data is visible
